# Plan Template

## Architecture
Describe the components, boundaries, and data flow needed to implement the spec.

## Design Decisions
- Record the important tradeoffs and the rationale behind them.
- Note any standards, conventions, or existing patterns that must be preserved.

## Dependencies
- List code, services, datasets, tools, or approvals required before implementation.
- Call out cross-team or cross-repository dependencies if they exist.

## Milestones
- Define the implementation milestones in delivery order.
- Include review, validation, and release checkpoints.
