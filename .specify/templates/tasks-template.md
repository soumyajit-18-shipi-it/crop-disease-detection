# Tasks Template

## Development Tasks
- Break the implementation into small, reviewable code changes.
- Include any migration, refactor, or wiring work required by the spec.

## Testing Tasks
- Add or update unit, integration, or regression tests for the feature.
- Verify edge cases and failure paths, not only the happy path.

## Documentation Tasks
- Update README, user guides, and inline comments where behavior changes.
- Add release notes or changelog entries when the change is user-visible.

## Deployment Tasks
- Describe validation steps for CI, packaging, Docker, or release automation.
- Include rollback or remediation steps if the deployment introduces risk.
